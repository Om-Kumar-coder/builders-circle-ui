COLLECTION 1: ownership_ledger
================================================================================
Collection ID: ownership_ledger
Collection Name: ownership_ledger
Description: Append-only ledger for tracking all ownership changes
Document Security: Enabled (row-level security)
Enabled: true

PERMISSIONS:
- read("users") - Any authenticated user can read

ATTRIBUTES:
┌─────────────────────┬──────────┬──────────┬──────────┬─────────┬─────────┐
│ Attribute Key       │ Type     │ Size     │ Required │ Array   │ Default │
├─────────────────────┼──────────┼──────────┼──────────┼─────────┼─────────┤
│ userId              │ string   │ 999      │ Yes      │ No      │ null    │
│ cycleId             │ string   │ 999      │ Yes      │ No      │ null    │
│ eventType           │ string   │ 999      │ Yes      │ No      │ null    │
│ ownershipAmount     │ double   │ -        │ Yes      │ No      │ null    │
│ multiplierSnapshot  │ double   │ -        │ Yes      │ No      │ null    │
│ scourceReference    │ string   │ 999      │ No       │ No      │ null    │
│ createdBy           │ string   │ 999      │ Yes      │ No      │ null    │
└─────────────────────┴──────────┴──────────┴──────────┴─────────┴─────────┘

ATTRIBUTE DETAILS:
- userId: Reference to Appwrite user ID who owns this ownership
- cycleId: Reference to build cycle ID
- eventType: Type of ownership event (e.g., "activity_submission", "multiplier_adjustment")
- ownershipAmount: Amount of ownership points (can be positive or negative)
- multiplierSnapshot: Multiplier value at time of event
- scourceReference: Optional reference to source document (e.g., activity ID)
- createdBy: User ID who created this entry (usually system or admin)

INDEXES:
1. userId_cycleId (key index, ASC)
   - Attributes: userId, cycleId
   - Purpose: Fast lookup of user's ownership in specific cycle

2. cycleId (key index, ASC)
   - Attributes: cycleId
   - Purpose: Fast lookup of all ownership in a cycle

3. userId (key index, ASC)
   - Attributes: userId
   - Purpose: Fast lookup of all user's ownership across cycles

BUSINESS RULES:
- Append-only: Records should never be updated or deleted
- Immutable audit trail for transparency
- All ownership changes must be logged here
- Used by computeOwnership function to calculate totals

- 0.5 = Diminishing (significantly reduced)

================================================================================
Collection ID: build_cycles

Enabled: true

PERMISSIONS:
- read("users") - Any authenticated user can read
- create("role:admin") - Only admins can create (recommended)
- update("role:admin") - Only admins can update (recommended)
- delete("role:admin") - Only admins can delete (recommended)

ATTRIBUTES:
┌─────────────────────┬──────────┬──────────┬──────────┬─────────┬─────────┐
│ Attribute Key       │ Type     │ Size     │ Required │ Array   │ Default │
├─────────────────────┼──────────┼──────────┼──────────┼─────────┼─────────┤
│ name                │ string   │ 255      │ Yes      │ No      │ null    │
│ state               │ string   │ 50       │ Yes      │ No      │ null    │
│ startDate           │ datetime │ -        │ Yes      │ No      │ null    │
│ endDate             │ datetime │ -        │ Yes      │ No      │ null    │
│ participantCount    │ integer  │ -        │ No       │ No      │ 0       │
└─────────────────────┴──────────┴──────────┴──────────┴─────────┴─────────┘

ATTRIBUTE DETAILS:
- name: Display name of the build cycle
- state: Current state of cycle (enum: planned, active, paused, closed)
- startDate: ISO 8601 datetime when cycle starts
- endDate: ISO 8601 datetime when cycle ends
- participantCount: Number of participants (optional, can be calculated)

INDEXES:
1. state (key index, ASC)
   - Attributes: state
   - Purpose: Fast filtering by cycle state

2. startDate (key index, DESC)
   - Attributes: startDate
   - Order: Descending (most recent first)
   - Purpose: Chronological ordering

BUSINESS RULES:
- Only one cycle should be "active" at a time (recommended)
- State transitions: planned → active → paused/closed
- Used by stallEvaluator and adjustMultiplier functions
- Frontend displays active cycles prominently

STATE VALUES:
- "planned" = Cycle is scheduled but not started
- "active" = Cycle is currently running
- "paused" = Cycle is temporarily paused
- "closed" = Cycle has ended

COLLECTION 4: cycle_participation
================================================================================
Collection ID: cycle_participation
Collection Name: cycle_participation
Description: Tracks user participation in build cycles
Document Security: Disabled (all authenticated users can see participation)
Enabled: true

PERMISSIONS:
- read("users") - Any authenticated user can read
- create("any") - Server/functions create records
- update("any") - Server/functions update records

ATTRIBUTES:
┌─────────────────────┬──────────┬──────────┬──────────┬─────────┬─────────┐
│ Attribute Key       │ Type     │ Size     │ Required │ Array   │ Default │
├─────────────────────┼──────────┼──────────┼──────────┼─────────┼─────────┤
│ userId              │ string   │ 999      │ Yes      │ No      │ null    │
│ cycleId             │ string   │ 999      │ Yes      │ No      │ null    │
│ optedIn             │ boolean  │ -        │ Yes      │ No      │ true    │
│ participationStatus │ string   │ 50       │ Yes      │ No      │ null    │
│ stallStage          │ string   │ 50       │ Yes      │ No      │ null    │
│ lastActivityDate    │ datetime │ -        │ No       │ No      │ null    │
└─────────────────────┴──────────┴──────────┴──────────┴─────────┴─────────┘

ATTRIBUTE DETAILS:
- userId: Reference to Appwrite user ID
- cycleId: Reference to build cycle ID
- optedIn: Whether user has opted into this cycle
- participationStatus: Current participation status (enum: active, at-risk, paused, grace)
- stallStage: Current stall stage (enum: none, grace, active, at_risk, diminishing, paused)
- lastActivityDate: ISO 8601 datetime of last activity submission (null if no activity yet)

INDEXES:
1. userId_cycleId (unique index, ASC)
   - Attributes: userId, cycleId
   - Type: Unique (one participation record per user per cycle)
   - Purpose: Ensure no duplicate participations

2. cycleId (key index, ASC)
   - Attributes: cycleId
   - Purpose: Fast lookup of all participants in a cycle

3. userId (key index, ASC)
   - Attributes: userId
   - Purpose: Fast lookup of all cycles a user participates in

4. stallStage (key index, ASC)
   - Attributes: stallStage
   - Purpose: Fast filtering by stall stage for reporting

BUSINESS RULES:
- One record per user per cycle (enforced by unique index)
- Created when user joins a cycle
- Updated by stallEvaluator function daily
- Updated when user submits activity

PARTICIPATION STATUS VALUES:
- "grace" = New participant, grace period (0 days)
- "active" = Actively participating (1-6 days since last activity)
- "at-risk" = Warning state (7-13 days inactive)
- "paused" = Participation paused (21+ days inactive)

STALL STAGE VALUES:
- "none" = No stall (not used currently)
- "grace" = Grace period (0 days, no activity yet)
- "active" = Active (1-6 days since last activity)
- "at_risk" = At risk (7-13 days inactive)
- "diminishing" = Diminishing (14-20 days inactive)
- "paused" = Paused (21+ days inactive)

STALL STAGE TO MULTIPLIER MAPPING:
- grace → 1.0
- active → 1.0
- at_risk → 0.75
- diminishing → 0.5
- paused → 0.0

================================================================================
APPWRITE FUNCTIONS THAT USE THESE COLLECTIONS
================================================================================

1. computeOwnership
   - Reads: ownership_ledger, multipliers
   - Writes: None
   - Purpose: Calculate total and effective ownership

2. stallEvaluator
   - Reads: build_cycles, cycle_participation
   - Writes: cycle_participation (updates stallStage, participationStatus)
   - Purpose: Evaluate participant inactivity daily

3. adjustMultiplier
   - Reads: build_cycles, cycle_participation, multipliers
   - Writes: multipliers, ownership_ledger
   - Purpose: Adjust multipliers based on stall stages daily
